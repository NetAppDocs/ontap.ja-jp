= ONTAP System ManagerによるMetroCluster IPサイト管理について学習します
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
MetroCluster構成では、別々の場所にある2つのONTAPクラスタ間でデータと設定を同期的にミラーリングします。ONTAP 9.8以降では、MetroCluster IP構成を管理するためのシンプルなインターフェイスとしてSystem Managerを使用できます。


NOTE: System Managerを使用してMetroCluster処理を実行できるのは、MetroCluster IP構成内のみです。MetroCluster FC構成でも、System Managerを使用してMetroCluster構成内の各ノードを管理することはできますが、MetroCluster固有の処理は実行できません。

通常は、2つの地理的に離れたサイトでMetroCluster構成の各クラスタをセットアップおよび設定します。その後、クラスタ間でデータが同期および共有されるようにピアリングをセットアップします。ピア ネットワークにある2つのクラスタは、それぞれがもう一方のクラスタのソース クラスタまたはバックアップ クラスタになることで、双方向のディザスタ リカバリ（DR）を実現します。8ノードまたは4ノードMetroCluster IP構成では、それぞれのサイトに、1つまたは2つのハイアベイラビリティ（HA）ペアを構成するストレージ コントローラが含まれます。

link:https://docs.netapp.com/us-en/ontap-metrocluster/install-ip/concept_mediator_requirements.html["ONTAP Mediatorをインストールする"^]を第3の場所に配置して、ノードとそのDRパートナーの状態を監視できます。ONTAP Mediatorは、災害発生時にMediator-Assisted Unplanned Switchover（MAUSO）を実行できます。

計画的なメンテナンスのために一方のクラスタを停止する場合には、ネゴシエート スイッチオーバーも実行できます。メンテナンスを実施したクラスタを起動してスイッチバック処理を実行するまで、両方のクラスタのすべてのデータI/O処理がパートナー クラスタで処理されます。

System Manager を使用した MetroCluster IP 構成のセットアップと管理の手順については、link:https://docs.netapp.com/us-en/ontap-metrocluster/index.html["MetroClusterのドキュメント"^]を参照してください。
