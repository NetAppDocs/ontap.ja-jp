= ONTAPコンシステンシグループを削除する
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
不要になった整合性グループは削除できます。

.タスク概要
* 整合性グループを削除すると、整合性グループのインスタンスが削除され、構成ボリュームまたはLUNには影響_しません_。整合性グループを削除しても、各ボリュームに存在するスナップショットは削除されませんが、整合性グループのスナップショットとしてアクセスできなくなります。ただし、これらのスナップショットは、通常のボリューム単位のスナップショットとして引き続き管理できます。
* 整合性グループ内のボリュームがすべて削除されると、整合性グループが自動的に削除されます。
* 親整合性グループを削除すると、関連付けられている子整合性グループがすべて削除されます。
* ONTAP 9.10.1～9.12.0のバージョンをご利用の場合、ボリューム自体を削除した場合にのみ、コンシステンシーグループからボリュームを削除できます。この場合、ボリュームはコンシステンシーグループから自動的に削除されます。ONTAP 9.12.1以降では、コンシステンシーグループ自体を削除せずに、コンシステンシーグループからボリュームを削除できます。このプロセスの詳細については、xref:modify-task.html[整合性グループの変更]を参照してください。


.手順
[role="tabbed-block"]
====
.System Manager
--
. *Storage > Consistency groups*を選択します。
. 削除する整合性グループを選択します。
. 整合性グループの名前の横にあるimage:../media/icon_kabob.gif["メニューオプションアイコン"]を選択し、次に*削除*を選択します。


--
.CLI
--
ONTAP 9.14.1以降では、CLIを使用して整合性グループを削除できます。

.開始する前に
* このタスクを実行するには、admin権限レベルが必要です。
* ONTAP 9.15.1以降では、admin権限レベルのすべてのユーザがこのタスクを実行できます。ONTAP 9.14.1では、このタスクを実行するには、クラスタ管理者またはSVMの管理者である必要があります。


.整合性グループの削除
. 整合性グループを削除します。
+
`consistency-group delete -vserver _svm_name_ -consistency-group _consistency_group_name_`
