= ONTAP NetAppボリューム暗号化を使用した外部キー管理の構成について学習します
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
1つ以上の外部キー管理サーバを使用して、暗号化されたデータにアクセスする際にクラスタで使用するキーを安全に保管できます。外部キー管理サーバはストレージ環境に配置されたサードパーティのシステムで、Key Management Interoperability Protocol（KMIP）を使用してノードにキーを提供します。オンボード キー マネージャに加えて、 ONTAP は複数の外部キー管理サーバをサポートしています。

ONTAP 9.10.1以降では、 xref:manage-keys-azure-google-task.html[Azure Key Vault または Google Cloud Key Manager サービス] データ SVM の NVE キーを保護します。ONTAP 9.11.1 以降では、クラスタ内に複数の外部キー マネージャを設定できます。見るxref:configure-cluster-key-server-task.html[クラスタ化されたキーサーバーを構成する]。ONTAP 9.12.0以降では、 link:https://docs.aws.amazon.com/kms/latest/developerguide/overview.html["AWS KMS"^] データ SVM の NVE キーを保護します。ONTAP 9.17.1以降では、OpenStackの xref:manage-keys-barbican-task.html[バービカンKMS] データ SVM の NVE キーを保護します。
