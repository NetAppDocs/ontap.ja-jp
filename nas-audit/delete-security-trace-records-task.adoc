= ONTAP SVMのセキュリティ トレース レコードを削除する
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
フィルター トレース レコードを使用してファイル アクセス セキュリティを確認したり、SMB または NFS クライアント アクセスの問題をトラブルシューティングしたりした後は、セキュリティ トレース ログからセキュリティ トレース レコードを削除できます。

.タスク概要
セキュリティ トレース レコードを削除する前に、レコードのシーケンス番号を知っておく必要があります。

[NOTE]
====
各Storage Virtual Machine（SVM）は、最大128件のトレースレコードを保存できます。SVMのトレースレコード数が上限に達した場合、新しいトレースレコードが追加されるたびに、最も古いトレースレコードが自動的に削除されます。このSVMのトレースレコードを手動で削除したくない場合は、ONTAPで上限に達した後に最も古いトレース結果を自動的に削除し、新しいトレース結果のためのスペースを確保することができます。

====
.手順
. 削除するレコードのシーケンス番号を特定します：
+
`vserver security trace trace-result show -vserver vserver_name -instance`

. セキュリティ トレース レコードを削除します：
+
`vserver security trace trace-result delete -node node_name -vserver vserver_name -seqnum integer`

+
`vserver security trace trace-result delete -vserver vs1 -node node1 -seqnum 999`

+
** `-node` `node_name`は、削除するパーミッション トレーシング イベントが発生したクラスター ノードの名前です。
+
これは必須パラメータです。

** `-vserver` `vserver_name`は、削除するパーミッション トレーシング イベントが発生した SVM の名前です。
+
これは必須パラメータです。

** `-seqnum` `integer` は、削除するログ イベントのシーケンス番号です。
+
これは必須パラメータです。
