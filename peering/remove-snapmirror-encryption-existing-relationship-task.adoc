= ピア関係からONTAPクラスタ ピアリング暗号化を削除する
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAP 9.6以降で作成されるすべてのピア関係では、デフォルトでクラスタ ピアリングの暗号化が有効になります。クラスタ間ピアリング通信に暗号化を使用しない場合は、暗号化を無効にできます。

.手順
. デスティネーション クラスタで、クラスタ ピアリングの暗号化を中止するようにソース クラスタとの通信を変更します。
+
** 認証を維持したまま暗号化を解除するには、次のコマンドを入力します。
+
[source, cli]
----
cluster peer modify <source_cluster> -auth-status-admin use-authentication -encryption-protocol-proposed none
----
** 暗号化と認証を解除するには、以下の手順に従います。
+
... 認証されていないアクセスを許可するようにクラスタ ピアリング ポリシーを変更します。
+
[source, cli]
----
cluster peer policy modify -is-unauthenticated-access-permitted true
----
... 暗号化および認証アクセスを変更します。
+
[source, cli]
----
cluster peer modify <source_cluster> -auth-status no-authentication
----




. プロンプトが表示されたら、パスフレーズを入力します。
. パスフレーズを再度入力して確認します。
. ソース クラスタで、デスティネーション クラスタとの通信の暗号化を無効にします。
+
** 認証を維持したまま暗号化を解除するには、次のコマンドを入力します。
+
[source, cli]
----
cluster peer modify <destination_cluster> -auth-status-admin use-authentication -encryption-protocol-proposed none
----
** 暗号化と認証を解除するには、以下の手順に従います。
+
... 認証されていないアクセスを許可するようにクラスタ ピアリング ポリシーを変更します。
+
[source, cli]
----
cluster peer policy modify -is-unauthenticated-access-permitted true
----
... 暗号化および認証アクセスを変更します。
+
[source, cli]
----
cluster peer modify <destination_cluster> -auth-status no-authentication
----




. プロンプトが表示されたら、デスティネーション クラスタで使用したパスフレーズを入力し、再度入力します。
