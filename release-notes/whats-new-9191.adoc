== SAN

[cols="30%,70%"]
|===
| 更新 | 概要 


 a| 
xref:../san-config/configure-fc-nvme-hosts-ha-pairs-reference.html[直接接続型FC構成のサポート]
 a| 
FCスイッチは、FCおよびFC-NVMeホストをONTAPストレージに接続するために必要ありません。ホストは、AFF、ASA、FASシステムのFCアダプタポートに直接接続できます。この直接接続方式は、簡便性が重要な遠隔地やハイブリッド環境において特に有用です。



 a| 
xref:../san-admin/multipath-automatic-lif-failover-support.html[AFF システムでのアクティブ / アクティブマルチパスのサポート]
 a| 
AFFシステムはSAN環境におけるアクティブ / アクティブマルチパスをサポートします。アクティブ / アクティブ構成では、LUNまたは名前空間を所有するHAペア内のすべてのパスがアクティブ最適化されるため、ストレージのフェイルオーバーが発生した場合でも、ホストはI/Oを再開するためにフェイルオーバーパスのALUAまたはANAの遷移を待つ必要がありません。これによりフェイルオーバーまでの時間が短縮され、SANのお客様は、以前はASAシステムでのみ利用可能であった同じ無停止パス回復力とロード バランシングを備えたAFFシステムでミッションクリティカルなワークロードを実行できるようになります。



 a| 
xref:../san-admin/multipath-automatic-lif-failover-support.html[SAN LIFの自動フェイルオーバーのサポートが強化されました]
 a| 
自動LIFフェイルオーバーは、AFFおよびASAシステムのiSCSIプロトコルLIFでアクティブ / アクティブおよびアクティブ / アクティブ / SCSIのみのマルチパス構成、およびNVMe/TCPプロトコルLIFでアクティブ / アクティブマルチパス構成でサポートされています。



 a| 
xref:../ontap-fli/get-started/learn-about-data-migration.html[iSCSIプロトコルの外部LUNインポートのサポート]
 a| 
Foreign LUN Import（FLI）は、バックエンド接続にiSCSIプロトコルをサポートしています。

== セキュリティ

[cols="30%,70%"]
|===
| 更新 | 概要 


 a| 
ARPワークロード特性がSystem Manager UIから削除されました
 a| 
System Managerでは、ベースラインや急増の統計情報を含む、Autonomous Ransomware Protection（ARP）のワークロード特性が表示されなくなりました。急増アクティビティが実際のランサムウェア攻撃や手動調整が必要な事象として誤って解釈されることが多かったため、この情報は削除されました。

ARPの動作と検出方法は変更されておらず、System Managerによるこれらの詳細情報の表示方法のみが影響を受けます。サージ統計は、高度な分析のためのONTAP CLIおよびREST APIオプションを使用して引き続き利用できます。



 a| 
xref:../anti-ransomware/use-cases-restrictions-concept.html#supported-configurations[Autonomous Ransomware Protection の SnapMirror 同期および SnapMirror アクティブ同期のサポート]
 a| 
ARP/AIは、サポートされているプラットフォーム上でSnapMirror同期（NASおよびSAN）およびSnapMirrorアクティブ同期SAN関係に参加するボリュームをサポートします。ARP/AIは、FAS、AFF、AFXシステム上のSnapMirror同期関係のプライマリボリューム、およびAFFとASA r2システム上のSnapMirrorアクティブ同期SAN関係のプライマリボリュームを保護できます。ARP/AIは、関係におけるアクティブな読み書き可能なプライマリボリューム上で動作し、プライマリ専用のARPスナップショットを作成し、機械学習ベースのランサムウェア検出を提供します。



 a| 
xref:../networking/configure-tls-offload.html[TLSオフロード設定]
 a| 
TLSオフロードを設定することで、サポートされているイーサネットカードのリソースを活用し、TLSハンドシェイク後のパフォーマンスを向上させることができます。この機能は暗号化と復号化の処理をオフロードすることで、CPU負荷を軽減し、パフォーマンスを向上させます。

|===


== ストレージ効率

[cols="30%,70%"]
|===
| 更新 | 概要 


 a| 
xref:../flexcache/index.html[NASパフォーマンスの強化]
 a| 
ONTAP 9.19.1では、小規模なNASディレクトリのリスト表示リクエストを処理するために、より多くのCPUリソースが利用できるようになりました。この機能強化は、2MB以下（約25,000ファイル）の小さなNASディレクトリを、数百から数千ものクライアントが同時にリスト表示している顧客にメリットをもたらします。

この機能強化により、ディレクトリ一覧表示を実行する多数のクライアントをサポートするために、ローカルFlexCacheボリュームなどの回避策を使用する必要性が軽減されます。クライアント数が少ない場合、パフォーマンスに悪影響はありません。このパフォーマンスの強化は自動的に行われるため、設定は不要です。

|===
