= ONTAP 9.19.1 からリバートする前にアクティブ / アクティブ SVM を削除する
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
SAN アクティブ / アクティブマルチパスは ONTAP 9.18.1 以前ではサポートされていません。ONTAP 9.19.1 からリバートする前に、アクティブ / アクティブ設定のストレージ仮想マシン（SVM）をすべて削除する必要があります。

.手順
. san-multipathing オプションが アクティブ / アクティブ または active-active-scsi-only に設定されている SVM を特定します：
+
[source, cli]
----
vserver show
----
. アクティブ / アクティブまたはアクティブ / アクティブ-scsi-only マルチパス用に設定されたすべての SVM を削除します：
+
[source, cli]
----
vserver delete -vserver <SVM>
----
. SVMが削除されていることを確認します。
+
[source, cli]
----
vserver show
----


.関連情報
ONTAP 9.19.1 以降の link:../san-admin/multipath-automatic-lif-failover-support.html["SAN アクティブ / アクティブ マルチパス対応"] の詳細をご確認ください。
