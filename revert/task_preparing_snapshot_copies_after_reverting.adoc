= ONTAP リバート後に Snapshot ポリシーを有効にする
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
以前のバージョンのONTAPにリバートした後、スナップショットの作成を再度開始するには、Snapshotポリシーを有効にする必要があります。

以前のバージョンのONTAPにリバートする前に無効にしたSnapshotスケジュールを再度有効にします。

.手順
. すべてのデータ SVM のスナップショットポリシーを有効にします：
+
[source, cli]
----
volume snapshot policy modify -vserver * -enabled true
----
+
[source, cli]
----
snapshot policy modify pg-rpo-hourly -enable true
----
. 各ノードに対して、ルートボリュームの Snapshot ポリシーを有効にします：
+
[source, cli]
----
run -node <node_name> vol options <volume_name> nosnap off
----
