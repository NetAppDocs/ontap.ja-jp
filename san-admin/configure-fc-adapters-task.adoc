= FCアダプタの設定
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
オンボードFCアダプタと一部のFC拡張アダプタカードは、イニシエーターポートまたはターゲットポートとして個別に設定できます。その他のFC拡張アダプタは、工場出荷時にイニシエーターまたはターゲットとして設定されており、変更できません。FC SFP+アダプタを搭載したサポート対象のUTA2カードを使用することで、追加のFCポートも利用できます。

イニシエータ ポートはバックエンド ディスク シェルフや、場合によっては外部ストレージ アレイに直接接続するために使用できます。ターゲット ポートは、FC スイッチへの接続にのみ使用できます。

FC用に設定されているオンボード ポートとCNA/UTA2ポートの数は、コントローラのモデルによって異なります。サポートされるターゲット拡張アダプタもコントローラ モデルによって異なります。ご使用のコントローラ モデルでサポートされているオンボードFCポートとターゲット拡張アダプタの完全なリストについては、link:https://hwu.netapp.com["NetApp Hardware Universe"^]を参照してください。



== FCアダプタのイニシエータ モード設定

イニシエータモードは、ポートをテープドライブ、テープライブラリ、またはlink:https://docs.netapp.com/us-en/ontap-fli/get-started/learn-about-san-data-migration.html["Foreign LUN Import（FLI）"^]を使用するサードパーティのストレージに接続するために使用されます。

.開始する前に
* アダプタのLIFを、メンバーとして属するすべてのポート セットから削除する必要があります。
* 変更対象の物理ポートを使用しているすべてのストレージ仮想マシン（SVM）から、すべてのLIFを移行または削除してから、物理ポートのパーソナリティをターゲットからイニシエータに変更する必要があります。


[NOTE]
====
NVMe/FCはイニシエータモードをサポートしていません。

====
.手順
. アダプタからすべてのLIFを削除します。
+
[source, cli]
----
network interface delete -vserver _SVM_name_ -lif _lif_name_,_lif_name_
----
. アダプタをオフラインにします。
+
[source, cli]
----
network fcp adapter modify -node _node_name_ -adapter _adapter_port_ -status-admin down
----
+
アダプタがオフラインにならない場合、システムの該当するアダプタ ポートからケーブルを取り外すこともできます。

. アダプタをターゲットからイニシエータに変更します。
+
[source, cli]
----
system hardware unified-connect modify -t initiator _adapter_port_
----
. 変更したアダプタをホストしているノードをリブートします。
. 構成に対してFCポートが正しい状態で設定されていることを確認します。
+
[source, cli]
----
system hardware unified-connect show
----
. アダプタをオンラインに戻します。
+
[source, cli]
----
node run -node _node_name_ storage enable adapter _adapter_port_
----




== FCアダプタのターゲット モード設定

ターゲット モードは、ポートをFCイニシエータに接続するために使用します。

FCアダプタをFCプロトコルとFC-NVMeプロトコル用に設定する手順は同じです。ただし、FC-NVMeをサポートしているのは一部のFCアダプタのみです。FC-NVMeプロトコルをサポートするアダプタの一覧については、link:https://hwu.netapp.com["NetApp Hardware Universe"^]をご覧ください。

.手順
. アダプタをオフラインにします。
+
[source, cli]
----
node run -node _node_name_ storage disable adapter _adapter_name_
----
+
アダプタがオフラインにならない場合、システムの該当するアダプタ ポートからケーブルを取り外すこともできます。

. アダプタをイニシエータからターゲットに変更します。
+
[source, cli]
----
system node hardware unified-connect modify -t target -node _node_name_ adapter _adapter_name_
----
. 変更したアダプタをホストしているノードをリブートします。
. ターゲット ポートの設定が正しいことを確認します。
+
[source, cli]
----
network fcp adapter show -node _node_name_
----
. アダプタをオンラインにします。
+
[source, cli]
----
network fcp adapter modify -node _node_name_ -adapter _adapter_port_ -state up
----




== FCアダプタをFCおよびFC-NVMeホストに直接接続するように設定します

ONTAP 9.19.1以降、FCおよびFC-NVMeホストとONTAPストレージ間の接続を確立するためにFCスイッチは必要ありません。ホストは、AFF、ASA、ASA r2、およびFASシステムのFCアダプタポートに直接接続できます。直接接続機能は、簡便性が重要な遠隔地やハイブリッド環境において特に有用です。FCポートをホストに直接接続する前に、FCポートをオフラインにしてトポロジーを変更する必要があります。

直接接続構成で使用されるFCポートには、以下の制限があります：

* このポートは、1つのStorage Virtual Machine（SVM）にのみ割り当てることができます。
* ポートが直接接続用に設定された後は、そのポートに割り当てることができるネットワークインターフェイス（LIF）は1つだけです。
* このポートはFCプロトコルまたはFC-NVMeプロトコルのいずれか一方のみをサポートし、両方をサポートすることはできません。


.開始する前に
クラスタ管理者である必要があります。

[role="tabbed-block"]
====
.System Manager
--
. System Manager で、* Network > FC ports * を選択します。
. 直接接続用に設定するポートの横にあるimage:icon_kabob.gif["選択アイコン"]を選択し、*トポロジーの編集*を選択します。
. *トポロジー*で*直接*を選択し、*トポロジーを変更してポートをオフラインにする*を選択します。
. * Save * を選択します。
. ターゲットポート用のケーブルをホストイニシエータに直接接続してください。
. ASA r2システムを使用している場合は、link:https://docs.netapp.com/us-en/asa-r2/administer/manage-client-vm-access.html#create-a-lif-network-interface["この System Manager の手順"^]に従ってLIFを作成してください。AFF、ASA、FASシステムを使用している場合は、次の手順に従ってLIFを作成してください：
+
.. 「ネットワーク」>「インターフェース」を選択し、次にimage:icon_add.gif["追加アイコン"]を選択します。
.. FCまたはNVMe/FCプロトコルを選択してください。
.. LIFに名前を付けるか、ONTAPによって生成された名前を使用します。
.. 少なくとも1つのサブネットが設定されている場合、LIFに使用するサブネットを選択できます。
.. サブネットを選択しない場合は、ブロードキャストドメインのIPアドレスとサブネットマスクを指定できます。
.. ブロードキャストドメインからホームポートを選択します。
+
ホームポート制御は、ブロードキャストドメインまたはサブネットの選択に基づいて表示されます。

.. *保存*を選択します。
+
LIFが作成されると、ポートは自動的にオンラインになります。





.次の手順
System Manager で、 * ネットワーク > FC ポート * を選択し、 * トポロジ * が * Direct * と表示されていることを確認します。

--
.CLI
--
. 直接接続に使用するポートが直接トポロジーをサポートしていることを確認します。
+
[source, cli]
----
network fcp adapter show -fields topologies-supported
----
. アダプタポートに使用されているLIFの管理ステータスと運用ステータスを表示します（：）
+
[source, cli]
----
network interface show -vserver <SVM_name>
----
. アダプタポートに使用されているLIFをオフラインにします。
+
[source, cli]
----
network interface modify -vserver <SVM_name> -lif* -status-admin down
----
. アダプタポートからすべてのLIFを削除します。
+
[source, cli]
----
network interface delete -vserver <SVM_name> -lif <lif_name>,<lif_name>
----
. アダプタをオフラインにします。
+
[source, cli]
----
network fcp adapter modify -node <node_name> -adapter <adapter_port> -status-admin down
----
. アダプタポートのトポロジをダイレクトに変更します：
+
[source, cli]
----
network fcp adapter modify -node <node_name> -adapter <adapter_port> -topology direct
----
. ターゲットポート用のケーブルをホストイニシエータに直接接続してください。
. 単一のLIFインターフェースを作成します：
+
[source, cli]
----
network interface create -vserver <SVM_name> -lif <lif_name> -data protocol fcp|fcp-nvme -home-node <node_name> -home-port <port_name>
----
+
LIFの作成後、ポートは自動的に有効になります。

. ポートが直接接続用に設定されていることを確認します。
+
[source, cli]
----
network fcp interface show
----
+
トポロジは直接接続として表示される必要があります。



--
====

== FCアダプタの速度を設定する

自動ネゴシエーションを使用するのではなく、アダプタのターゲット ポートの速度を接続先デバイスの速度に合わせて設定する必要があります。自動ネゴシエーションに設定されたポートは、テイクオーバー/ギブバックなどの中断後に再接続するまでに時間がかかる場合があります。

.タスク概要
このタスクはクラスタ内のすべてのStorage Virtual Machine（SVM）とすべてのLIFを対象としているため、 `-home-port`パラメータと `-home-lif`パラメータを使用して、この操作の範囲を制限する必要があります。これらのパラメータを使用しない場合、操作はクラスタ内のすべてのLIFに適用されるため、望ましくない可能性があります。

.開始する前に
このアダプタをホームポートとして使用するすべてのLIFはオフラインである必要があります。

.手順
. このアダプタ上のすべての LIF をオフラインにします：
+
[source, cli]
----
network interface modify -vserver * -lif * { -home-node node1 -home-port 0c } -status-admin down
----
. アダプタをオフラインにします。
+
[source, cli]
----
network fcp adapter modify -node node1 -adapter 0c -state down
----
+
アダプタがオフラインにならない場合、システムの該当するアダプタ ポートからケーブルを取り外すこともできます。

. ポート アダプタの最大速度を確認します。
+
[source, cli]
----
fcp adapter show -instance
----
+
アダプタの速度を最大速度を超えて変更することはできません。

. アダプタの速度を変更します：
+
[source, cli]
----
network fcp adapter modify -node node1 -adapter 0c -speed 16
----
. アダプタをオンラインにします：
+
[source, cli]
----
network fcp adapter modify -node node1 -adapter 0c -state up
----
. アダプタ上のすべての LIF をオンラインにします：
+
[source, cli]
----
network interface modify -vserver * -lif * { -home-node node1 -home-port 0c } -status-admin up
----
