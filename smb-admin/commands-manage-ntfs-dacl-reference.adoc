= SMBサーバ上のNTFS DACLアクセス制御エントリを管理するためのONTAPコマンド
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
DACL アクセス制御エントリ（ACE）を管理するための ONTAP 専用コマンドがあります。NTFS DACL にはいつでも ACE を追加できます。また、DACL 内の ACE に関する情報を変更、削除、表示することで、既存の NTFS DACL を管理することもできます。

|===
| 状況 | 使用するコマンド 


 a| 
ACEを作成し、NTFS DACLに追加する
 a| 
`vserver security file-directory ntfs dacl add`



 a| 
NTFS DACL 内の既存の ACE を変更する
 a| 
`vserver security file-directory ntfs dacl modify`



 a| 
NTFS DACL 内の既存の ACE に関する情報を表示する
 a| 
`vserver security file-directory ntfs dacl show`



 a| 
NTFS DACLから既存のACEを削除する
 a| 
`vserver security file-directory ntfs dacl remove`

|===
 `vserver security file-directory ntfs dacl`の詳細については、link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=vserver+security+file-directory+ntfs+dacl["ONTAPコマンド リファレンス"^]をご覧ください。
