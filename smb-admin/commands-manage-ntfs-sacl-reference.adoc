= SMBサーバ上のNTFS SACLアクセス制御エントリを管理するためのONTAPコマンド
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
SACL アクセス制御エントリ（ACE）を管理するための ONTAP 専用コマンドがあります。NTFS SACL にはいつでも ACE を追加できます。また、SACL 内の ACE に関する情報を変更、削除、表示することで、既存の NTFS SACL を管理することもできます。

|===
| 状況 | 使用するコマンド 


 a| 
ACEを作成し、NTFS SACLに追加する
 a| 
`vserver security file-directory ntfs sacl add`



 a| 
NTFS SACLの既存のACEを変更する
 a| 
`vserver security file-directory ntfs sacl modify`



 a| 
NTFS SACL内の既存のACEに関する情報を表示する
 a| 
`vserver security file-directory ntfs sacl show`



 a| 
NTFS SACLから既存のACEを削除する
 a| 
`vserver security file-directory ntfs sacl remove`

|===
 `vserver security file-directory ntfs sacl`の詳細については、link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=vserver+security+file-directory+ntfs+sacl["ONTAPコマンド リファレンス"^]をご覧ください。
