= ONTAP の NTFS および SMB のセキュリティポリシーとアクセス制御エントリを管理するためのコマンド
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
ONTAPコマンドを使用して、DACLおよびSACL内のNTFSセキュリティ記述子とアクセス制御エントリ（ACE）に関する情報を作成、変更、削除、表示できます。また、ONTAPコマンドを使用して、SMBセキュリティポリシー、ポリシータスク、ポリシージョブを管理することもできます。



[[ontap-commands-for-managing-ntfs-security-descriptors-on-smb-servers]]
== SMBサーバー上のNTFSセキュリティ記述子を管理するためのONTAPコマンド

セキュリティ記述子を管理するための専用のONTAPコマンドがあります。セキュリティ記述子に関する情報を作成、変更、削除、表示できます。

|===
| 状況 | 使用するコマンド 


 a| 
NTFSセキュリティ記述子を作成する
 a| 
`vserver security file-directory ntfs create`



 a| 
既存のNTFSセキュリティ記述子を変更する
 a| 
`vserver security file-directory ntfs modify`



 a| 
既存の NTFS セキュリティ記述子に関する情報を表示する
 a| 
`vserver security file-directory ntfs show`



 a| 
NTFSセキュリティ記述子を削除する
 a| 
`vserver security file-directory ntfs delete`

|===
 `vserver security file-directory ntfs`の詳細については、link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=vserver+security+file-directory+ntfs["ONTAPコマンド リファレンス"^]をご覧ください。



[[ontap-commands-for-managing-ntfs-dacl-access-control-entries-on-smb-servers]]
== SMBサーバ上のNTFS DACLアクセス制御エントリを管理するためのONTAPコマンド

DACL アクセス制御エントリ（ACE）を管理するための ONTAP 専用コマンドがあります。NTFS DACL にはいつでも ACE を追加できます。また、DACL 内の ACE に関する情報を変更、削除、表示することで、既存の NTFS DACL を管理することもできます。

|===
| 状況 | 使用するコマンド 


 a| 
ACEを作成し、NTFS DACLに追加する
 a| 
`vserver security file-directory ntfs dacl add`



 a| 
NTFS DACL 内の既存の ACE を変更する
 a| 
`vserver security file-directory ntfs dacl modify`



 a| 
NTFS DACL 内の既存の ACE に関する情報を表示する
 a| 
`vserver security file-directory ntfs dacl show`



 a| 
NTFS DACLから既存のACEを削除する
 a| 
`vserver security file-directory ntfs dacl remove`

|===
 `vserver security file-directory ntfs dacl`の詳細については、link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=vserver+security+file-directory+ntfs+dacl["ONTAPコマンド リファレンス"^]をご覧ください。

セキュリティポリシーを削除する

[[ontap-commands-for-managing-smb-security-policy-jobs]]
== SMBセキュリティポリシージョブを管理するためのONTAPコマンド

セキュリティ ポリシー ジョブの一時停止、再開、停止、および情報表示を行うための ONTAP コマンドがあります。

|===
| 状況 | 使用するコマンド 


 a| 
セキュリティ ポリシー ジョブを一時停止する
 a| 
`vserver security file-directory job pause ‑vserver vserver_name -id integer`



 a| 
セキュリティポリシージョブを再開する
 a| 
`vserver security file-directory job resume ‑vserver vserver_name -id integer`



 a| 
セキュリティ ポリシー ジョブに関する情報を表示する
 a| 
`vserver security file-directory job show ‑vserver vserver_name` このコマンドを使用してジョブのジョブ ID を判別できます。



 a| 
セキュリティポリシージョブを停止する
 a| 
`vserver security file-directory job stop ‑vserver vserver_name -id integer`

|===
 `vserver security file-directory job`の詳細については、link:https://docs.netapp.com/us-en/ontap-cli/search.html?q=vserver+security+file-directory+job["ONTAPコマンド リファレンス"^]をご覧ください。
