= ONTAP SMB 自動ノード紹介のサポート
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
SMB 自動ノード リファーラルを有効にする前に、特定の ONTAP 機能がリファーラルをサポートしていないことに注意する必要があります。

* 次の種類のボリュームは、SMB 自動ノード リファーラルをサポートしていません：
+
** 負荷共有ミラーの読み取り専用メンバー
** データ保護ミラーの宛先ボリューム


* ノード リファーラルは LIF の移動と一緒には移動しません。
+
クライアントが SMB 2.x または SMB 3.0 接続を介して参照された接続を使用しており、データ LIF が無停止で移動した場合、LIF がデータに対してローカルではなくなったとしても、クライアントは同じ参照された接続を引き続き使用します。

* ノード リファーラルはボリュームの移動と一緒には移動しません。
+
クライアントが SMB 接続を介して参照された接続を使用しているときにボリュームの移動が発生すると、ボリュームがデータ LIF と同じノード上に存在しなくなっても、クライアントは同じ参照された接続を引き続き使用します。
