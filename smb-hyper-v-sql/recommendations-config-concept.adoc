= SQL ServerおよびHyper-V over SMB構成に関する推奨事項
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
SQL Server および Hyper-V over SMB 構成が堅牢で動作可能であることを確認するには、ソリューションを構成する際に推奨されるベスト プラクティスをよく理解しておく必要があります。



== 一般的な推奨事項

* アプリケーション サーバー ファイルを一般ユーザー データから分離します。
+
可能であれば、ストレージ仮想マシン（SVM）とそのストレージ全体をアプリケーション サーバーのデータ専用にします。

* 最適なパフォーマンスを得るには、アプリケーション サーバのデータを保存するために使用されるSVMでSMB署名を有効にしないでください。
* 最高のパフォーマンスとフォールトトレランスの向上を実現するには、SMBマルチチャネルを有効にして、単一のSMBセッションでONTAPとクライアント間の複数の接続を提供します。
* Hyper-VまたはSQL Server over SMB構成で使用する共有以外では、継続的な可用性が確保された共有を作成しないようにします。
* 継続的可用性に使用される共有で変更通知を無効にします。
* アグリゲートの再配置（ARL）と同時にボリュームの移動を実行しないでください。ARLには一部の処理を一時停止するフェーズがあります。
* Hyper-V over SMBソリューションでは、クラスタ化された仮想マシンを作成する際にゲスト内のiSCSIドライブを使用してください。共有 `.VHDX`ファイルは、ONTAP SMB共有内のHyper-V over SMBではサポートされません。
