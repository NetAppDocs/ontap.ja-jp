= ONTAP 整合グループまたは NAS SVM 間の共通スナップショットを作成する
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
定期的に実行されるスナップショット操作に加えて、SAN構成のプライマリ SnapMirror 整合性グループのボリュームとセカンダリ SnapMirror 整合性グループのボリューム間、またはNAS構成のSVMレベルで、共通のlink:../concepts/snapshot-copies-concept.html["Snapshot"]を手動で作成することもできます。

.タスク概要
自動フェイルオーバーポリシーの場合、スケジュールされたスナップショット作成間隔は 12 時間、自動フェイルオーバーデュプレックスポリシーの場合は 3 時間です。

.開始する前に
* SnapMirrorグループ関係が同期されている必要があります。


.手順
System Manager または ONTAP CLI を使用して、共通スナップショットを作成できます。

[role="tabbed-block"]
====
.System Manager
--
. デスティネーション クラスタから、*Protection > Replication* に移動します。
. 関係ソースを見つけて、image:icon_kabob.gif["メニューオプションアイコン"] を選択し、*更新*を選択します。
. *[ポリシーに従って]* または *[スナップショットを選択]* を選択して、既存のスナップショットを選択します。
. *Update* を選択します。


--
.CLI
--
. デスティネーション サイトから、共通のスナップショットを作成します。
+
`snapmirror update -destination-path <destination path>`

+
SAN構成の例：

+
[listing]
----
destination::>snapmirror update -destination-path vs1_dst:/cg/cg_dst
----
+
NAS構成の例：

+
[listing]
----
snapmirror update -destination-path vs_b:
----
. 更新の進行状況を監視します。
+
`destination::>snapmirror show -fields newest-snapshot`
