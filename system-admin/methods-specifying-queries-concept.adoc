= ONTAP CLIでクエリ演算子を使用する
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
管理インターフェイスでは、クエリおよびUNIX形式のパターンとワイルドカードがサポートされており、コマンド パラメータ引数の複数の値を照合できます。

次の表に、サポートされるクエリ演算子を示します。

[cols="10,90"]
|===
| オペレーター | 概要 


 a| 
*
 a| 
すべてのエントリに一致するワイルドカード。

たとえば、コマンド `volume show -volume \*tmp*`は、名前に文字列 `tmp`が含まれるすべてのボリュームのリストを表示します。



 a| 
!
 a| 
NOT演算子。

一致しない値を示します。たとえば、 `*!vs0*`は値 `vs0`と一致しないことを示します。



 a| 
|
 a| 
OR演算子。

比較する2つの値を区切ります。例えば、 `*vs0 | vs2*`vs0またはvs2のいずれかに一致します。複数のORステートメントを指定できます。例えば、 `a | b* | \*c*`はエントリ `a`、 `b`で始まる任意のエントリ、 `c`を含む任意のエントリに一致します。



 a| 
..
 a| 
範囲演算子。

たとえば、 `*5..10*`は `5`から `10`までの任意の値と一致します。



 a| 
<
 a| 
小なり演算子。

たとえば、 `*<20*`は `20`より小さい任意の値と一致します。



 a| 
>
 a| 
大なり演算子。

たとえば、 `*>5*`は `5`より大きい任意の値と一致します。



 a| 
\<=
 a| 
以下演算子。

たとえば、 `*<=5*`は `5`以下の任意の値と一致します。



 a| 
>=
 a| 
以上演算子。

たとえば、 `*>=5*`は `5`以上の任意の値と一致します。



 a| 
{`query`}
 a| 
拡張クエリ。

拡張クエリは、コマンド名に続けて、他のパラメータの前に最初の引数として指定する必要があります。

たとえば、このコマンド `volume modify {-volume \*tmp*} -state offline`は、名前に文字列 `tmp`が含まれるすべてのボリュームをオフラインに設定します。

|===
クエリ文字をリテラルとして解析する場合は、正しい結果が返されるように、文字を二重引用符で囲む必要があります（たとえば、 `"<10"`、 `"0..100"`、 `"\*abc*"`、または `"a|b"`）。

特殊文字が解釈されないように、rawファイル名は二重引用符で囲む必要があります。これは、クラスタシェルで使用される特殊文字にも当てはまります。

1つのコマンドラインで複数のクエリ演算子を使用できます。例えば、このコマンド `volume show -size >1GB -percent-used <50 -vserver !vs1`は、サイズが1GBを超え、使用率が50%未満で、「`vs1`」という名前のStorage Virtual Machine（SVM）に存在しないすべてのボリュームを表示します。
