= ONTAP System ManagerでNFSを使用してLinuxサーバー用のNASストレージを有効にする
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Linuxクライアントにデータを提供するために、Storage VMを作成または変更してNFSサーバを有効にします。

この手順を使用して、新規または既存のStorage VMでNFSプロトコルを有効にできます。

image:workflow_nas_enable_linux_nfs.png["ワークフローの概要：1 NFS を有効にする 2 ルートボリュームのエクスポートポリシーを開く 3 DNS を構成する 4 ネームサービスを構成する 5 必要に応じて暗号化を構成する"]

.開始する前に
環境に必要なネットワーク、認証、セキュリティのサービスについて、設定の詳細を忘れないようにメモしておきます。

.手順
. Storage VMでNFSを有効にします。
+
** 新しいストレージ VM の場合：* ストレージ > ストレージ VM * を選択し、* 追加 * を選択してストレージ VM 名を入力し、* SMB/CIFS、NFS、S3 * タブで * NFS を有効にする * を選択します。
+
... デフォルトの言語を確定します。
... ネットワーク インターフェイスを追加します。
... Storage VMの管理者アカウントの情報を更新します（オプション）。


** 既存のストレージ VM の場合：*クラスター > ストレージ VM* を選択し、ストレージ VM を選択し、*設定* を選択してから、*NFS* の下にある image:icon_gear.gif["アクションアイコン"] または image:icon-edit-pencil-blue-outline.png["編集アイコン"] を選択し、*Enable NFS* を選択します。
+
... 必要に応じて、特定のNFSプロトコルバージョンとオプションを有効または無効にします。
+

NOTE: FPolicyによるNFSv4.2ファイルアクセスイベントの監視はサポートされていません。FPolicy監視を設定する場合は、SVM上でNFSv4.2を無効にする必要があります。参考までに、link:nfs-admin/enable-disable-nfs-task.html["ONTAP SVM の NFS を有効または無効にする"]を参照してください。





. Storage VMルート ボリュームのエクスポート ポリシーを開きます。
+
.. *Storage > Volumes* を選択し、Storage VM のルートボリューム（デフォルトでは _volume-name_ _root）を選択してから、*Export Policy* の下に表示されるポリシーを選択します。
.. *追加*を選択してルールを追加します。
+
*** クライアント仕様 = `0.0.0.0/0`
*** アクセス プロトコル = NFS
*** アクセスの詳細 = UNIX読み取り専用




. ホスト名解決のための DNS の設定：*ストレージ > ストレージ VM* を選択し、ストレージ VM を選択し、*設定* を選択してから、*DNS* の下の image:icon_gear.gif["アクションアイコン"] を選択します。
. ネーム サービスを必要に応じて設定します。
+
.. 「ストレージ」>「ストレージ VM」を選択し、ストレージ VM を選択し、「設定」を選択してから、image:icon_gear.gif["アクションアイコン"] LDAP または NIS を選択します。
.. 名前サービス切り替えタイルでimage:icon_pencil.gif["編集アイコン"]を選択して、変更を含めます。


. 必要に応じて、Kerberosを設定します。
+
.. * Storage > Storage VMs *を選択し、ストレージVMを選択してから、* Settings *を選択します。
.. Kerberosタイルでimage:icon_arrow.gif["矢印アイコン"]を選択し、*追加*を選択します。
