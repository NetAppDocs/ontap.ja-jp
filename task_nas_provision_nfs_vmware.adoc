= ONTAP System Managerを使用してVMware データストア用のNFSストレージをプロビジョニングする
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Virtual Storage Console for VMware vSphere（VSC）を使用してONTAPベースのストレージ システム上のNFSボリュームをESXiホスト用にプロビジョニングする場合は、事前にONTAP 9.7以降のSystem Managerを使用してNFSを有効にします。

System Manager でlink:task_nas_enable_linux_nfs.html["NFS対応ストレージVM"]を作成した後、NFS ボリュームをプロビジョニングし、VSC を使用してデータストアを管理します。

VSC 7.0 以降、VSC は https://docs.netapp.com/us-en/ontap-tools-vmware-vsphere/index.html["ONTAP Tools for VMware vSphere 仮想アプライアンス"^]の一部となり、これには VSC、vStorage APIs for Storage Awareness（VASA）Provider、および Storage Replication Adapter（SRA）for VMware vSphere機能が含まれます。

必ず https://imt.netapp.com/matrix/["NetApp Interoperability Matrix"^]を確認して、現在の ONTAP と VSC リリース間の互換性を確認してください。

System Manager Classic（ONTAP 9.7以前のリリース）を使用してESXiホストからデータストアへのNFSアクセスを設定するには、 https://docs.netapp.com/us-en/ontap-system-manager-classic/nfs-config-esxi/index.html["VSC を使用した ESXi の NFS 設定の概要"^]を参照してください。

詳細については、 https://docs.netapp.com/us-en/netapp-solutions/virtualization/vsphere_ontap_ontap_for_vsphere.html["TR-4597：VMware vSphere for ONTAP"^]および VSC リリースのドキュメントを参照してください。
