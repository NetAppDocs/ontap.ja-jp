= 自動更新の有効化
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
System Manager を使用して自動更新を有効にし、ONTAP でユーザーの操作なしにファームウェア、システム、およびセキュリティのアップデートをダウンロードしてインストールできるようにします。

ONTAP CLIを使用してSP/BMCファームウェアの更新のみを管理できます。

* link:../system-admin/methods-manage-sp-bmc-firmware-updates-concept.html["ONTAP SP/BMC ファームウェアのアップデートの管理"]の詳細をご覧ください。
* System Manager と CLI による自動更新管理の違いについては、ナレッジベースの記事をご覧ください：link:https://kb.netapp.com/on-prem/ontap/Ontap_OS/OS-KBs/Why_do_GUI_and_CLI_show_different_status_for_ONTAP_automatic_updates_configuration["GUIとCLIでONTAP自動更新設定のステータスが異なって表示されるのはなぜですか？"]


各システム コンポーネントの自動更新の可用性は ONTAP バージョンによって異なります。

[cols="25,75"]
|===
| ONTAPのバージョン | 利用可能な自動更新 


| 9.16.1以降  a| 
* 人工知能を利用した自律型ランサムウェア対策（ARP / AI）
* ONTAP Time Zone Database
* ストレージ デバイス、ディスク、およびディスク シェルフのストレージ ファームウェア
* サービス プロセッサおよびBMCモジュールのSP / BMCファームウェア
* ディスク認定パッケージ（DQP）




| 9.13.1以降  a| 
* ONTAP Time Zone Database
* ストレージ デバイス、ディスク、およびディスク シェルフのストレージ ファームウェア
* サービス プロセッサおよびBMCモジュールのSP / BMCファームウェア
* ディスク認定パッケージ（DQP）




| 9.10.1以降  a| 
* ストレージ デバイス、ディスク、およびディスク シェルフのストレージ ファームウェア
* サービス プロセッサおよびBMCモジュールのSP / BMCファームウェア
* ディスク認定パッケージ（DQP）


|===
お使いのバージョンのONTAPで自動更新が利用できない場合、または自動更新が有効になっていない場合は、ファームウェア、Time Zone Database、およびセキュリティの更新を手動で実行できます。

.タスク概要
自動更新を有効にするには、EULA の条件に同意し、環境に応じて各システム コンポーネントのデフォルトの更新動作を確認または設定する必要があります。

ONTAP 9.16.1 以前で作成され、その後 9.16.1 以降にアップグレードされたクラスタの場合：

* アップグレード前に自動更新機能が有効になっている場合、アップグレード後もシステムコンポーネントの更新のデフォルト設定は変更されません。
* アップグレード前に自動更新が有効になっていない場合、ONTAP 9.16.1以降のシステムコンポーネント更新のデフォルト設定が適用されます。


システムコンポーネントの自動更新のデフォルト設定は、ONTAPバージョンに応じて、自動的に更新するか通知を表示するかのいずれかです。手順を完了する前に、これらの設定が環境に適切であることを確認してください。

この https://www.youtube.com/watch?v=GoABILT85hQ["ビデオ"^]では、自動更新プロセスの使用に関する簡単な概要を示します。

.開始する前に
* 有効なサポート資格が必要です。link:https://mysupport.netapp.com/site/["NetAppサポートサイト"^]の*システム詳細*ページでご確認いただけます。
* 自動更新を有効にするには、まずHTTPSでAutoSupportを有効にする必要があります。クラスタでAutoSupportが有効になっていない場合、または別のトランスポート プロトコルでクラスタのAutoSupportが有効になっている場合は、この手順中にHTTPSで有効にできます。



NOTE: HTTPS転送プロトコルを使用してテクニカル サポートにメッセージを送信するよう設定している場合は、AutoSupport OnDemandがデフォルトで有効になっており、機能しています。

* ONTAP 9.10.1 以降、自動更新を有効にする場合は、次の追加 URL への HTTPS 接続があることを確認してください：
+
** \https://support-sg-naeast.netapp.com
** \https://support-sg-nawest.netapp.com




.手順
[role="tabbed-block"]
====
.System Manager - ONTAP 9.16.1以降
--
. System Managerで、*Cluster > Settings*を選択します。
. HTTPS で AutoSupport OnDemand が有効になっていない場合は、image:icon_kabob.gif["メニューオプションアイコン"] を選択して、続行するために必要な設定を有効にします。
. *ソフトウェア アップデート* セクションで、*有効化* を選択します。
. 更新タイプごとに実行する操作を指定します。
+
更新タイプごとに、自動的に更新するか、通知を表示するか、自動的に更新を却下するかを選択できます。

. 利用規約に同意して、*保存*を選択します。


--
.System Manager - ONTAP 9.15.1 以前
--
. System Managerで、*Events*を選択します。
. *概要* セクションの *自動更新を有効にする* の横にある *アクション > 有効にする* を選択します。
. HTTPSを使用したAutoSupportを有効化していない場合は、選択して有効にします。
. 利用規約に同意して、*保存*を選択します。


--
.CLI
--
ONTAP CLIを使用してSP/BMCファームウェアの更新を管理できるのは次の場合のみです

. 自動ファームウェア更新を有効にする：
+
[source, cli]
----
system service-processor image modify -node <node_name> -autoupdate true
----
